    Running Mocha Tests with Webpack in Browser¶

    Intro¶

    This document describes running mocha tests in the browser, using webpack to bundle.

    Working repo on github: https://github.com/craigphicks/mocha-webpack-template

    By Mode:

    1. Production mode
    2. Served using express
    3. Development mode
    4. Served using webpacks own internal development browser, and not using express.

    Step by step:

    • Production mode
    • using a single test file
    • using a multiple test files
    • using a multiple test files and node dependencies
      • tree pruning vs tree shaking
    • Development mode

    top

 |-app

     |-favicon.ico

    Server File¶

    server.ts

    mport express from 'express';
const port = 3080

const app = express();

app.use('/',express.static('../static'))

app.listen(port, '127.0.0.1', () => {
    console.log(`http://localhost:${port}`);
})

    How to compile this express server not covered here.

    Single Test File¶

    index.ts

    export function test(){
  describe('test', () => {
    it('1/3 passes ', () => {
      if (false) throw new Error('fail');
    });
    it('2/3 fails', () => {
      if (true) throw new Error('fail');
    });
    it('3/3 passes', () => {
      if (false) throw new Error('fail');
    });
  });  
}

    Multiple Test Files¶

    Must be bundled into one file using a bundler such as Webpack.

    index.ts

b = false;

    test.js

    function throwIfNot(b:boolean){if (!b) throw new Error('fail');}
export function test(b:boolean){
  describe('test', () => {
    it('1/3 passes ', () => {
      throwIfNot(true)
    });
    it('2/3 ???', () => {
      throwIfNot(b)
    });
    it('3/3 passes', () => {
      throwIfNot(true)
    });
  });  
}

    From the app directory 

    npx run clean && npx run build

    Assuming the express server is running visit http://localhost:3080

    Lodash dependency and tree shaking vs tree pruning¶

    test.js

    import * as _ from 'lodash';
function throwIfNot(b:boolean){if (!b) throw new Error('fail ');}
export function test(b:boolean){
  describe('test', () => {
    it('1/4 passes ', () => {
      throwIfNot(true)
    });
    it('2/4 passes', () => {
      const s = _.join(['1','2'],'')  
      throwIfNot(s==='12')
    });
    it('3/3 ???', () => {
      throwIfNot(b)
    });
    it('3/4 passes', () => {
      throwIfNot(true)
    });
  });  
}

    1/4 passes

    About the big 77k size.

    As per https://webpack.js.org/guides/tree-shaking/ 

    {
  sideEffect:false
}

    is added to package.json, but it makes no difference to the size (yet).

    In test.js change inport * as _ from 'lodash' to import {join} from 'lodash.
    However, this still does not help.

    Instead try import join from 'lodash/join' Success! It works.

    $ ls -al ../static/index.js
-rw-rw-r-- 1 craig craig 772 Jun  1 16:49 ../static/index.js

    test.js

    import join from 'lodash/join'
function throwIfNot(b: boolean) {
  if (!b) throw new Error('fail ');
}
export function test(b: boolean) {
  describe('test', () => {
    it('1/4 passes ', () => {
      throwIfNot(true);
    });
    it('2/4 passes', () => {
      const s = join(['1', '2'], '');
      console.log(s);
      throwIfNot(s === '12');
    });
    it('3/3 ???', () => {
      throwIfNot(b);
    });
    it('3/4 passes', () => {
      throwIfNot(true);
    });
  });
}

    Apparently lodash-es is just lodash in ES6 format, which should allow tree-shaking rather than manual tree-pruning. However, using lodash-es seems no difference - manual pruning is still required.

    Development mode - hot module update and webpack server¶

    Hot module update enables the browser content to be updated automatically as a result of changing any source file, including any typescript test file.

    Webpack offers it own built in HTML server for this, so we won't be using express.

    The webpack.mocha.config and package.json files are modifed to handle both modes - production and development.

    webpack.mocha.config

    const path = require('path');

const DIST = path.resolve(__dirname, '../static');
module.exports = (env) => {
  console.log(`env=${JSON.stringify(env, null, 2)}`);
  const isdev = env.dev;
  let ret = {
    mode: isdev ? 'development' : 'production',
    entry: ['./test-in-js'],
    output: {
      filename: 'index.js',
      path: DIST,
    },
  };
  if (isdev) {
    ret = {
      ...ret,
      watch: true,
      devServer: {
        contentBase: DIST,
        watchContentBase: true,
        // proxy: {
        //   '/api': 'http://localhost:3080',
        // },
      },
      devtool: 'inline-source-map',
    };
  }
  console.log('config='+JSON.stringify(ret,null,2))
  return ret
};
